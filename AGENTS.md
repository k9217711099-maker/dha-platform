# AGENTS.md

Гид по проекту для Codex. Составлен на основе ТЗ `Промт.docx`.

## Что это за проект

Единая цифровая платформа для гостей сети **D Hotels & Apartments (D H&A)**:
мобильные приложения iOS/Android, веб-версия личного кабинета на сайте,
собственный модуль бронирования (booking engine), программа лояльности,
онлайн-регистрация гостя, цифровой ключ через TTLock.

Бизнес-цель: рост доли **прямых** бронирований, повторных проживаний и среднего
чека, снижение нагрузки на администраторов.

Статус: проектирование/старт разработки. Кода пока нет — документ ТЗ и набор
дизайн-скиллов в `skills/`.

## Ключевой архитектурный принцип (не нарушать)

```
Гость → мобильное приложение / сайт → backend D H&A → внешние системы
                                          ├─ Bnovo PMS
                                          ├─ Bitrix24
                                          ├─ TTLock
                                          ├─ платёжный провайдер
                                          └─ SMS / email / push
```

**Клиенты (мобайл/веб) НИКОГДА не обращаются напрямую к Bnovo, Bitrix24 или
TTLock.** Все внешние интеграции идут только через backend D H&A. Причины:
безопасность API-ключей, контроль бизнес-логики, единая база гостей, независимая
программа лояльности, защита ПДн, возможность сменить PMS в будущем.

### Источники истины
- **Доступность и тарифы** → Bnovo PMS.
- **Программа лояльности** → backend D H&A (собственная система, не зависит от Bnovo/Bitrix24).
- **Коммуникации и задачи** → Bitrix24 (CRM и рабочее место сотрудников).

## Рекомендуемый стек

- **Мобайл:** React Native или Flutter.
- **Backend:** Node.js / NestJS либо Python / Django / FastAPI.
- **БД:** PostgreSQL. **Кэш:** Redis. **Очереди:** RabbitMQ / BullMQ / Celery.
- **Файлы:** S3-совместимое хранилище (сканы документов — с повышенной защитой).
- **Веб:** личный кабинет + booking engine, адаптивная вёрстка (мобайл + десктоп).

Стек ещё не зафиксирован окончательно — это рекомендации ТЗ. Уточнить выбор у
владельца перед генерацией бойлерплейта.

## Доменные правила, которые легко нарушить

### Программа лояльности (валюта — «баллы D»)
- 1 балл = 1 ₽ при списании.
- Баллы начисляются **только за оплаченные и завершённые прямые бронирования**
  (сайт, приложение, звонок, мессенджер, прямое бронирование через менеджера).
  За OTA-бронирования баллы можно не начислять.
- Не начисляются за отменённые бронирования; база начисления — стоимость
  проживания без налогов/депозитов/штрафов/возвратов/услуг партнёров.
- Уровни: **Member 3%** (регистрация) → **Silver 5%** (50 000 ₽ или 5 ночей / 12 мес,
  списание до 15%) → **Gold 7%** (150 000 ₽ или 15 ночей / 12 мес) →
  **Platinum 10%** (300 000 ₽ или 30 ночей / 12 мес). Пороги считаются по прямым
  бронированиям за скользящие 12 месяцев.
- Срок действия баллов: 24 месяца, продлевается ещё на 24 при новом прямом бронировании.
- Мин. списание 500 баллов. Баллами нельзя оплачивать депозит и штрафы.
- Статусы баллов: ожидают начисления / доступны / списаны / заморожены / сгорели / отменены.
  Баллы становятся «доступны» только после выезда и подтверждения оплаты.
- Часть привилегий (ранний заезд/поздний выезд/апгрейд) доступна **только в отелях
  сети, не в апартаментах/квартирах** — учитывать тип объекта.

### Цифровой ключ TTLock
- Ключ создаётся только если: бронь активна и не отменена, наступило разрешённое
  время выдачи, пройдена онлайн-регистрация, выполнены оплата/депозит (если требуется).
- Окно действия по умолчанию: за 30 мин до заезда — через 30 мин после выезда
  (настраивается в админ-панели). **После проживания пароль/доступ удаляется.**
- Типы доступа: PIN, Bluetooth eKey, через gateway, резервный код администратора,
  доступ для персонала. Логировать весь жизненный цикл ключа.

### Бронирования
- Раздел «Мои бронирования» делится строго на 4: текущие, предстоящие, прошлые,
  отменённые — у каждого свой набор полей и действий (см. ТЗ §7).
- При создании брони backend обязан: проверить актуальность цены/доступности в
  Bnovo → создать бронь в Bnovo → начислить/зарезервировать баллы → отправить
  подтверждение → показать в ЛК.
- Синхронизация с Bnovo: API + webhooks (если есть) + fallback-поллинг каждые
  5–15 мин, с журналом ошибок и ретраями.

## Объём работ (MVP vs v2)

аналитика, сегментация, интеграции с платежами и фискализацией.

## Безопасность и ПДн

HTTPS, JWT/OAuth2 + refresh tokens, 2FA для администраторов, журнал действий
админов, разграничение прав. Паспортные данные и сканы документов — повышенный
уровень защиты: шифрование файлов, ограничение доступа, срок хранения, удаление по
регламенту, логирование просмотров.
